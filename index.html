E-commerce y Dashboard para "INSOMNIO"

Como Encargado de Software en MYTHOUGHTSEC, lideré el desarrollo de una solución completa de e-commerce para la marca de moda "INSOMNIO".

El proyecto abarcó desde la creación de una tienda online atractiva y funcional para el cliente final, hasta un potente panel de administración (dashboard) para la gestión interna. El dashboard permite administrar colecciones, categorías, productos, usuarios y órdenes. Además, ofrece analíticas de ventas clave, como ventas totales, nuevos usuarios y estado de las órdenes, para facilitar la toma de decisiones estratégicas.

Tecnologías Clave:

  • Frontend: React / Next.js, Tailwind CSS
  • Backend: Node.js, Express
  • Base de Datos: MongoDB / PostgreSQL
  • Despliegue: Railway, AWS

NexusERP - Sistema de Inventario

Desarrollo de un sistema de gestión de inventarios (ERP) a medida para la empresa de flores AFL-ALE Flowers LLC. El objetivo fue digitalizar y optimizar su control logístico.

La plataforma permite un seguimiento en tiempo real de productos, control de existencias, gestión de lotes y fechas de vencimiento, y configuración de alertas automáticas. El sistema cuenta con diferentes roles de usuario y un dashboard centralizado que ofrece una visión clara del estado del inventario, reduciendo errores y mejorando la eficiencia operativa.

Tecnologías Clave:

  • Frontend: NextJS
  • Backend: NestJS
  • Base de Datos: PostgreSQL

Flara Nexus - Gestión Ágil con GitHub

Se estructuró el ciclo de vida del desarrollo utilizando GitHub Projects para crear un flujo de trabajo transparente y eficiente. Software de Gestión Dental (Argentina)

Lideré la fase de consultoría, planificación y diseño de la arquitectura para un software de gestión integral (SaaS) para una red de clínicas dentales en Argentina.

FicaAssistant: Chatbot para la UTN

Creación de un asistente virtual inteligente para la Facultad de Ingeniería en Ciencias Aplicadas (FICA) de la Universidad Técnica del Norte.

El proyecto utilizó un modelo de lenguaje grande (LLM) tipo LLaMA, afinado específicamente con la normativa oficial de la universidad. El proceso incluyó la recolección de documentos (PDF, Word), su conversión y limpieza mediante OCR para crear un corpus textual, y finalmente, el entrenamiento del modelo para sintetizar la información y responder con precisión a las consultas de los estudiantes sobre temas como cambios de carrera, requisitos y procedimientos.

Componentes de IA:

  • Modelo Base: LLaMA
  • Procesamiento: OCR, Limpieza y Normalización de Texto
  • Técnica: Fine-tuning (Afinamiento)
  • Aplicación: Procesamiento de Lenguaje Natural (NLP)
